# Spec Format — `specs/`

Specs are written **before** implementing features. Follow this structure:

## Required sections

1. **Title** — `# [Feature Name] — Spec` or `# Especificação: [Feature Name]`
2. **Context / Visão geral** — What the feature does and current state
3. **Specification** — Technical details (schema, components, APIs, etc.)
4. **TODOs** — Checklist (`- [ ]`) for implementation steps

## Optional (when relevant)

- **Research** — How others solve it (links, patterns)
- **Alternatives** — Comparison table before deciding
- **Decision** — Why the chosen approach was selected
- **Code reference** — Snippets for schema, config, etc.

## Conventions

- Use `---` to separate major sections
- Tables for enums, columns, comparisons
- Code blocks for schema, config, snippets
- Keep TODOs actionable and ordered
